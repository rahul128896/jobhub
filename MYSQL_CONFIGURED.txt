TROUBLESHOOTING
================================================================================

Issue: "MySQL Connection Error"
Solution: 
  1. Verify MySQL is running
  2. Check password in backend/.env (Rahul@123)
  3. Run: python scripts/test_mysql.py

Issue: "Unknown database 'jobhub'"
Solution:
  1. Run: python scripts/setup_mysql.py
  2. Or manually: mysql -u root -p < schema.sql

Issue: "Table doesn't exist"
Solution:
  1. Start app (auto-creates tables): python app.py
  2. Or run setup script again

Issue: "Connection refused"
Solution:
  1. Start MySQL service
  2. Windows: net start MySQL80
  3. Linux/Mac: sudo service mysql start


IMPORTANT NOTES
================================================================================

1. The .env file contains sensitive credentials
   - Never commit to public repositories
   - Add to .gitignore in production
   
2. Database backups
   - Backup: mysqldump -u root -p jobhub > backup.sql
   - Restore: mysql -u root -p jobhub < backup.sql
   
3. For production
   - Change MySQL password
   - Use strong SECRET_KEY
   - Set FLASK_ENV=production
   - Configure SSL/TLS
   - Set up automated backups
